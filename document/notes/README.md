# CFBox 工作记录（批级）

> 这里放**批级 / 单次有意义迭代**的工作记录，文件名 `<YYYY-MM-DD>-<topic>.md`。参考既有记录风格（如本目录首篇 `2026-06-17-ai-infra-migration.md`）。
>
> **与 [changelogs/](../../changelogs/) 分工**：本目录是**开发过程**记录（一批一记，背景/目标/设计/决策/陷阱/验证）；`changelogs/` 是**发版级**总结（v0.1.0/v0.2.0…，面向用户的功能与体积变化）。一批属于某版本时，过程进本目录，版本汇总进 changelogs。

## 推荐结构

```markdown
# <标题>（<日期>，<批/Phase 归属>）

## 背景
为什么做这批改动（问题/动机）。

## 目标
这批要达成什么（可验证的完成门）。

## 设计 / 决策
关键设计选择与权衡（含被否方案的理由）。

## 陷阱（GOTCHA）
踩过的坑、跨批须注意的不变量（可回填到 PLAN.md 的 OPEN GOTCHAS）。

## 验证
跑了什么（`ctest`/`run_all.sh`/size-opt），结果（测试数、体积）。

## 后续
遗留项、下一批衔接。
```

> 非正式 diff 复述；写**为什么**和**决策**，给未来回看的人（含 AI）上下文。
